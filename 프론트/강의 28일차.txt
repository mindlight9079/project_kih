웹의 기본 목적
- 다른 여러 컴퓨터에서 문서를 공유하거나 보는 목적

웹의 구성
- 웹 서버와 웹 클라이언트 컴퓨터들로 구성
  - 웹 서버
	웹 문서, 이미지, 동영상 등의 데이터 저장 관리   * CDN에 이미지파일을 접근할 수 있도록 관리
       	웹 클라이언트의 요청을 받아 웹 문서 전송
 
  - 웹 클라이언트


클라이언트가 요청을 하면 서버가 작업을 처리한 후에 처리한 결과를 웹한테 전달한다.


웹 브라우저의 종류
 -인터넷 익스플로러
 - 파이어폭스
 - 오페라
 - 크롬

웹 사이트 구축
 - 웹 서보로 사용할 컴퓨터에 웹 서버 소프트웨어 설치
 - 웹 페이지, 동영상, 이미지 저장, 데이터베이스 설치
 - 웹 서버 응용프로그램 개발 및 설치


웹 페이지의 주소, URL
 - 프로토콜, 서버주소, TCP/IP 포트번호, 경로명, 웹페이지 파일이름
   http://, www.oracle.com, :80, /technetwrok/java, index.html
	- 프로토콜 : HTTP, https, file, ftp, telnet, mailto, news 등
	- 서버주소 : 웹 페이지를 가진 컴퓨터의 인터넷 주소, IP주소
	- TCP/IP 포트번호 : 서버가 브라우저로부터 접속을 기다리는 TCP/IP 포트 번호.
			프로토콜마다 다르다.


웹 페이지 구성
- 웹 페이지 구성 3요소
	- 웹페이지의 구조와 내용 - HTML
	- 웹 페이지의 모양 - CSS(Cascading Style Sheet)
	- 웹 페이지의 행동 및 응용 프로그램 - Javascript => jQuery
- 웹 페이지는 3 요소를 분리하여 개발


HTML5 출현 배경
1. 비표준 기술의 혼재, 웹 브라우저의 비 호환성.
2. 인터넷 기기의 다양화
	- 기존의 웹페이지가 모바일 기기에서 동작하지 않음.
3. 새로운 범용 웹 표준의 필요성



HTML 페이지의 필수 태그
<!DOCTYPE html> - HTML5 문서임을 알리는 지시어
<html>, <head>, <title>,<body> 태그


HTML 태그 구성
< img src = "heart.jpg" >
img = 태그이름
src = "heart.jpg" = 속성


title : 브라우저 탭 상단에 제목을 나타내는 태그
h? : 제목태그(h1~h6)
