하나의 프로그램 : 하나의 프로세스 + 스레드(여러개)


컬렉션 프레임워크
  -자료들을 효율적으로 관리하기 위한 것
  - List, Set, Map : 인터페이스
  - List와 Set은 한 종류의 데이터를 관리, Map은 2종류의 데이터를 관리
  - List : 순서 보장, 중복 가능
  - Set : 중복 불가능, 순서를 보장하지 않음.
  - Map : key와 value로 이루어짐. key는 중복 불가. value는 중복 가능.


List : 인터페이스
  - 순서 보장, 중복 가능.
  - ArrayList  
	º List의 구현 클래스
	º 배열로 된 리스트
	º 탐색이 빠르다.
	º 중간 삽입/삭제가 느리다.
  - LinkedList
	º List의 구현 클래스
	º 연결로 된 리스트
	º 탐색이 느리다.
	º 삽입/삭제가 빠르다.


Set : 인터페이스
  - 중복 불가능, 순서 보장x
  - HashSet
  	º Hash 코드를 이용하여 먼저 확인한 후 equals()를 이용하여 확인

Map
  - key와 value로 이루어짐
  - key는 중복 불가, value는 중복 가능
  - 출력할 때 조금 복잡
  - HashMap
	º Map의 구현 클래스
